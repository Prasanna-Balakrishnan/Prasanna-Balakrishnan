## ⚙️ How I Approach Problems

My workflow is simple:

1. Understand the problem clearly  
2. Build a minimal correct solution  
3. Test edge cases  
4. Fix logic, not symptoms  
5. Refine until explanation and behavior align  

If I can’t explain a solution cleanly, I don’t consider it complete.  
If it works on the first try, I double-check it.
